WSL2 搭建 Linux 开发环境（零基础教程）

适用场景：Windows 10/11 系统下搭建 Linux（Ubuntu）开发环境，适配 C++/Python/Shell 等开发需求，无需双系统/传统虚拟机。

一、WSL2 简介

WSL 全称是 Windows Subsystem for Linux，中文常称为“适用于 Linux 的 Windows 子系统”。它可以让我们在 Windows 中直接运行 Linux 发行版，例如 Ubuntu、Debian、Kali 等，并使用 Bash、APT、GCC、Python、Git、Make、CMake 等常见 Linux 工具。

WSL2 是第二代 WSL，相比 WSL1，它使用真实的 Linux 内核，并通过轻量级虚拟化技术运行 Linux 环境。因此，它对 Linux 系统调用的兼容性更好，也更适合用来进行 C++、Python、Shell、后端服务、嵌入式交叉编译等开发工作。

  • 不用装双系统：保留 Windows 的日常使用体验，同时获得 Linux 命令行环境。
  • 比传统虚拟机更轻量：启动快、资源占用相对更低，适合日常开发。
  • Windows 与 Linux 文件互通：Linux 中可访问 Windows 磁盘，Windows 中也能打开 WSL 文件。
  • 适合开发者：可以直接安装 Linux 下的软件包，方便进行编译、调试、运行脚本和部署测试。

二、前置条件

  • Windows 10 版本 2004+ 或 Windows 11（按 Win+R 输入 winver 查看版本）。
  • 建议开启 CPU 虚拟化功能。可在“任务管理器 → 性能 → CPU”中查看“虚拟化”是否已启用。
  • 保持网络连接正常，因为安装 Ubuntu 时需要联网下载发行版文件。

三、步骤 1：启用 WSL 功能

方式 1：命令行快速开启（推荐）

以管理员身份打开 PowerShell，执行以下命令：

wsl --install

命令说明：

  • 自动启用「适用于 Linux 的 Windows 子系统」和「虚拟机平台」功能。
  • 如果电脑之前没有安装过 WSL，它会自动安装 WSL 相关组件。
  • 执行后如果系统提示重启，请先重启电脑，再继续后面的步骤。

方式 2：手动开启（备用）

  1. 按 Win+I 打开设置 → 应用 → 可选功能 → 更多 Windows 功能。
  2. 勾选「适用于 Linux 的 Windows 子系统」和「虚拟机平台」。
  3. 点击「确定」，等待安装完成后重启电脑。

四、步骤 2：设置 WSL2 为默认版本

管理员身份打开 PowerShell，执行：

wsl --set-default-version 2

为了避免 WSL 组件过旧，也可以顺手更新 WSL：

wsl --update

如果提示「WSL 2 需要更新其内核组件」，可以先执行 wsl --update，再重启电脑后继续。

五、步骤 3：安装 Ubuntu 22.04

安装 Ubuntu 有两种常用方法：一种是通过 Microsoft Store 安装，适合初学者；另一种是不使用微软商店，直接通过 PowerShell 命令安装，适合想快速部署环境或遇到商店下载异常的情况。

方法 1：通过 Microsoft Store 安装 Ubuntu（图形界面）

  1. 打开 Microsoft Store。
  2. 在搜索框中输入 Ubuntu 22.04。
  3. 找到 Ubuntu 22.04 LTS 后，点击“获取”或“安装”。
  4. 安装完成后，点击“打开”，等待 Ubuntu 首次初始化。
  5. 根据提示设置 Linux 用户名和密码。

这种方式操作直观，适合不熟悉命令行的用户。如果 Microsoft Store 打不开、下载速度慢、登录异常，建议使用下面的命令行方式。

方法 2：不使用 Microsoft Store，使用命令安装 Ubuntu 22.04（推荐）

以管理员身份打开 PowerShell，先查看当前系统支持安装哪些 Linux 发行版：

wsl --list --online

该命令会列出当前可安装的发行版名称，例如 Ubuntu、Ubuntu-22.04、Debian 等。

然后执行下面命令安装 Ubuntu 22.04：

wsl --install -d Ubuntu-22.04
注意：推荐使用 wsl --install -d Ubuntu-22.04。其中 --install 是两个短横线；发行版名称建议写成 Ubuntu-22.04。如果写成 wsl -install -d ubuntu22.04，部分系统可能无法识别命令或发行版名称。

命令行安装失败时的备用方式

如果安装过程中出现 Microsoft Store 连接失败、下载卡住、网络异常等情况，可以尝试加入 --web-download 参数，改为从网页渠道下载安装：

wsl --install --web-download -d Ubuntu-22.04

检查 Ubuntu 是否安装成功

无论采用 Microsoft Store 还是命令行方式安装，完成后都建议在 PowerShell 中执行：

wsl -l -v

正常情况下会看到类似结果：

  NAME            STATE           VERSION
* Ubuntu-22.04    Running         2
  • NAME：发行版名称。
  • STATE：运行状态，可能是 Running 或 Stopped。
  • VERSION：WSL 版本，显示 2 代表正在使用 WSL2。

如果 VERSION 显示为 1，可以执行下面命令转换为 WSL2：

wsl --set-version Ubuntu-22.04 2

六、步骤 4：启动 Ubuntu 并初始化用户

安装完成后，可以直接在 PowerShell 中启动 Ubuntu：

wsl -d Ubuntu-22.04

首次启动会初始化 Linux 文件系统，然后要求你设置 Linux 用户名和密码：

  • 用户名可以和 Windows 用户名不同，建议使用英文小写。
  • 输入密码时终端不会显示星号或圆点，这是 Linux 的正常安全机制。
  • 这个密码主要用于执行 sudo 管理员命令，请自己记住。

七、步骤 5：配置开发环境（C++ 为例）

在 Ubuntu 终端中执行以下命令，安装开发依赖：

# 更新软件源
sudo apt update && sudo apt upgrade -y

# 安装 C++ 开发工具链
sudo apt install gcc g++ cmake make git -y

# 验证安装
gcc --version
g++ --version
cmake --version
git --version

八、步骤 6：常用 WSL 命令（必备）

  • 查看 WSL 版本信息：wsl --version
  • 查看 WSL 当前状态：wsl --status
  • 查看已安装的发行版：wsl --list --verbose 或 wsl -l -v
  • 查看可安装的发行版：wsl --list --online
  • 启动默认 WSL：PowerShell 输入 wsl
  • 启动指定发行版：wsl -d Ubuntu-22.04
  • 关闭所有 WSL 实例：wsl --shutdown
  • 设置默认发行版：wsl --set-default Ubuntu-22.04
  • 卸载指定发行版：wsl --unregister Ubuntu-22.04（会删除该 Ubuntu 内的所有文件，慎用）
小技巧：WSL2 中访问 Windows 文件：路径为 /mnt/c/Users/你的Windows用户名/（C盘）、/mnt/d/（D盘）；Windows 访问 WSL 文件：在文件资源管理器输入 \\wsl$\Ubuntu-22.04。

九、步骤 7：推荐工作目录

虽然 WSL 可以直接访问 Windows 磁盘，但建议把 Linux 项目放在 WSL 自己的家目录中，例如：

mkdir -p ~/projects
cd ~/projects

原因是 Linux 文件系统下的编译、安装依赖和权限管理通常更稳定，尤其适合 C++、Python、Node.js、Go 等项目。

十、常见问题解决

  • 执行 wsl --install 只显示帮助信息：检查 Windows 版本是否过低，或先执行 wsl --update 后重试。
  • 启动 WSL 提示 0x800701bc：通常是 WSL2 内核组件过旧，执行 wsl --update 后重启电脑。
  • Microsoft Store 打不开或下载失败：可以改用 wsl --install -d Ubuntu-22.04 命令安装。
  • 命令行安装 Ubuntu 22.04 失败：先运行 wsl --list --online 确认可用发行版名称，再重新安装。
  • 下载卡住或商店连接失败：尝试 wsl --install --web-download -d Ubuntu-22.04。
  • 检查发现 Ubuntu 是 WSL1：执行 wsl --set-version Ubuntu-22.04 2 转换到 WSL2。
  • Ubuntu 终端中文乱码：执行 sudo dpkg-reconfigure locales，选择 zh_CN.UTF-8 并设为默认。
  • 权限不足：命令前加 sudo，并输入你设置的 Linux 密码。
总结：最核心的安装流程可以记为：启用 WSL → 设置默认 WSL2 → 选择 Microsoft Store 或命令行安装 Ubuntu 22.04 → 使用 wsl -l -v 检查版本 → 进入 Ubuntu 安装开发工具链。
